      20                 :            :  *
      21                 :            :  * A SegMate process group is a QE (Query Executor) Writer process and 0, 1 or
      22                 :            :  * more QE Reader processes. Greenplum needed to invent a SegMate sharing
      23                 :            :  * mechanism because in Postgres there is only 1 backend and most needed
      24                 :            :  * information is simply available in private memory. With Greenplum session
      25                 :            :  * parallelism on database instances, we need to have a way to share
      26                 :            :  * not-yet-committed session information among the SegMates. This information
      27                 :            :  * includes transaction snapshots, sub-transaction status, so-called combo-cid
      28                 :            :  * mapping, etc.
      29                 :            :  *
      30                 :            :  * An example: the QE readers need to use the same snapshot and command number
      31                 :            :  * information as the QE writer so they see the current data written by the QE
      32                 :            :  * writer. During a transaction, the QE Writer writes new data into the
      33                 :            :  * shared-memory buffered cache. Later in that same transaction, QE Readers
      34                 :            :  * will need to recognize which tuples in the shared-memory buffered cache are
      35                 :            :  * for its session's transaction to perform correctly.
      36                 :            :  *
      37                 :            :  * Another example: the QE readers need to know which sub-transactions are
      38                 :            :  * active or committed for a session's transaction so they can properly read
      39                 :            :  * sub-transaction data written by the QE writer for the transaction.
      40                 :            :  *
      41                 :            :  * So, the theme is to share private, not-yet-committed session transaction
      42                 :            :  * information with the QE Readers so the SegMate process group can all work
      43                 :            :  * on the transaction correctly. [We mostly think of QE Writers/Readers being
      44                 :            :  * on the segments. However, masters have special purpose QE Reader called the
      45                 :            :  * Entry DB Singleton. So, the SegMate module also works on the master.]

      48                 :            :  * Writer gang member establishes a local transaction, acquires the slot in
      49                 :            :  * shared snapshot shmem space and copies the snapshot information into shared
      50                 :            :  * memory where the other qExecs that are segmates can find it. Following
      51                 :            :  * section convers details on how shared memory initialization happens, who
      52                 :            :  * writes the snapshot, how its controlled how/when the readers can read the
