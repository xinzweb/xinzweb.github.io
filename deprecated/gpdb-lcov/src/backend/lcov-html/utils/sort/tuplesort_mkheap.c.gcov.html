      18                 :            : /*
      19                 :            :  * Multi-Key Heap.
      20                 :            :  *
      21                 :            :  * This is a heap, plus an array.  The array (called lvtops) holds the first N keys of the top heap element.
      22                 :            :  *        N is the entry's level
      23                 :            :  *        For other entries, their level indicates for how many of those N keys they match the top heap element
      24                 :            :  *
      25                 :            :  * For example, a heap with 3 multi-key entries:
      26                 :            :  *        ['bob','barker','impostor']           <== this is the top entry of the heap
      27                 :            :  *        ['bob','barker','price is right']
      28                 :            :  *        ['bob','evans', 'sausage']
      29                 :            :  *
      30                 :            :  * So, our lvtops could have:
      31                 :            :  *        ['bob','barker']
      32                 :            :  *
      33                 :            :  *        And our entries would have the levels:
      34                 :            :  *                 2: top entry of the heap so indicates # of values in lvtops
      35                 :            :  *                 2: matches the top entry/lvtops on the first two keys ('bob' and 'barker')
      36                 :            :  *                 1: matches the top entry/lvtops on first key ('bob')
      37                 :            :  *
      38                 :            :  *        Note that lvtops could also have the following, with the given entries
      39                 :            :  *        ['bob','barker','impostor']
      40                 :            :  *                 3: top entry of the heap so indicates # of values in lvtops
      41                 :            :  *                 2: matches the top entry/lvtops on the first two keys ('bob' and 'barker')
      42                 :            :  *                 1: matches the top entry/lvtops on first key ('bob')
      43                 :            :  *
      44                 :            :  *      Therefore, level tells you something useful about the relationship between values.
      45                 :            :  *
      46                 :            :  *      Note also that because we CANNOT insert values SMALLER than the top of the heap,
      47                 :            :  *         the level of an entry (say, e) never goes down -- because that would require lvtops
      48                 :            :  *         values to become smaller which is impossible without adding a smaller value!
      49                 :            :  *
      50                 :            :  * The purpose of this weird heap is that:
      51                 :            :  *        1) we do not do any unnecessary lv expansions -- meaning, many tuples in a multi-key comparison differ
      52                 :            :  *               only in first few keys so no need to materialize the later keys and comparable keys.
      53                 :            :  *        2) the compare is cheap because once you've determined the level of a row then you can compare using only level
      54                 :            :  *
      55                 :            :  * Testing note: see attachment on MPP-7231 for some useful testing code
      56                 :            :  *
      57                 :            :  *
      58                 :            :  *
      59                 :            :  * NOTE ON UNIQUENESS CHECKING:
      60                 :            :  *
      61                 :            :  * When required to enforce uniqueness, by necessity we can only do that either when:
      62                 :            :  *        a) a value is inserted and we detect that it equals the root then uniqueness is violated
      63                 :            :  *        b) at any point, if one of the root's children exactly matches the root then uniqueness is violated
      64                 :            :  *
      65                 :            :  * However, when inserting a value that is greater than the root then we can't check whether it is
      66                 :            :  *        equal to some other value in the heap.  Therefore, the heap may contain duplicate values at
