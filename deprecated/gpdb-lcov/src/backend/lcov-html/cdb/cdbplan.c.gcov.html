      36                 :            : /*--------------------
      37                 :            :  * expression_tree_mutator() is designed to support routines that make a
      38                 :            :  * modified copy of an expression tree, with some nodes being added,
      39                 :            :  * removed, or replaced by new subtrees.  The original tree is (normally)
      40                 :            :  * not changed.  Each recursion level is responsible for returning a copy of
      41                 :            :  * (or appropriately modified substitute for) the subtree it is handed.
      42                 :            :  * A mutator routine should look like this:

      44                 :            :  * Node * my_mutator (Node *node, my_struct *context)
      45                 :            :  * {

      58                 :            :  *              return expression_tree_mutator(node, my_mutator, (void *) context);
      59                 :            :  * }

      62                 :            :  * information the mutator routine needs --- it can be used to return extra

      64                 :            :  * expression_tree_mutator, but it is passed down to recursive sub-invocations
      65                 :            :  * of my_mutator.  The tree walk is started from a setup routine that
      66                 :            :  * fills in the appropriate context struct, calls my_mutator with the
      67                 :            :  * top-level node of the tree, and does any required post-processing.
      68                 :            :  *
      69                 :            :  * Each level of recursion must return an appropriately modified Node.
      70                 :            :  * If expression_tree_mutator() is called, it will make an exact copy
      71                 :            :  * of the given Node, but invoke my_mutator() to copy the sub-node(s)

      73                 :            :  * copying process but need not directly deal with expression trees
